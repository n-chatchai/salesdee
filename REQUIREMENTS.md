## 1. ภาพรวมผลิตภัณฑ์

### 1.1 มันคืออะไร
แพลตฟอร์ม **SaaS แบบ multi-tenant** สำหรับงานก่อนการขาย (presale) + CRM ของธุรกิจ SME ไทย:
รับลูกค้าเข้ามา (lead) จากหลายช่องทาง → จัดการ pipeline การขาย → ออกใบเสนอราคาที่สวยและถูกต้องตามแบบไทย → ส่ง/ติดตามผ่าน LINE/อีเมล → ปิดการขาย → ส่งต่องานหลังการขาย (ออเดอร์/ผลิต/ส่ง/ติดตั้ง/รับประกัน) → (เฟสถัดไป) ออกใบกำกับภาษี/ใบเสร็จ + ลูกหนี้ + บัญชีพื้นฐาน

**Beachhead = ธุรกิจเฟอร์นิเจอร์** (ดีลเลอร์/ผู้ผลิต/ร้านเฟอร์นิเจอร์สำนักงาน-บ้าน-บิวท์อิน-โครงการ) — ออกแบบฟีเจอร์ใบเสนอราคา/แคตตาล็อกให้ "รู้เรื่องเฟอร์นิเจอร์" ตั้งแต่ต้น แล้วจึงค่อยขยายไป SME แนวขายโครงการ/ขายโซลูชันอื่น ๆ

ลูกค้าอ้างอิงรายแรก (anchor / dogfooding): บริษัท วัน.ดี.ดี.บิสซิเนส จำกัด (wandeedeefurniture) — เฟอร์นิเจอร์สำนักงานขายให้องค์กร (ออฟฟิศ/โรงแรม/โรงเรียน), รับประกัน 1 ปี, ติดต่อผ่าน LINE/WhatsApp/โทรเป็นหลัก

### 1.2 ปัญหาที่แก้ให้ SME
- ลูกค้าเข้ามาทาง LINE/โทร/เพจ แล้ว**หลุดมือ** เพราะไม่มีที่เก็บ ไม่มีคนตามต่อ
- ออกใบเสนอราคาช้า/ไม่สวย/ผิดพลาด (ลืม VAT, ตัวเลขเป็นตัวอักษร, เลขที่ซ้ำ, แก้ราคาแล้วสับสนเวอร์ชัน)
- ไม่รู้ว่ามีดีลค้างกี่ดีล มูลค่าเท่าไหร่ ใครต้องตามใคร เซลส์คนไหนทำได้แค่ไหน
- ของเฟอร์นิเจอร์: ใบเสนอราคามีหลายห้อง หลายวัสดุ หลายรูป จัดการในไฟล์ Excel/Word ลำบาก
- เก็บเงินเป็นงวด (มัดจำ/ก่อนติดตั้ง/งานเสร็จ) ตามยาก, ลูกค้าค้างชำระไม่รู้ตัว

### 1.3 Positioning — เป็นอะไร / ไม่ใช่อะไร
- **เป็น**: Presale + CRM + Quotation (เฟสแรก) → Order/Delivery/Warranty tracking (เฟส 2) → Invoice/Tax Invoice/AR + บัญชีพื้นฐาน (เฟส 3) — Thai-first
- **ไม่ใช่**: ระบบบัญชีเต็มรูป (ไม่มีวงจรซื้อ/AP, สินทรัพย์ถาวร, งบกระแสเงินสด, ภ.พ.30/ภ.ง.ด. ครบชุด, งบรวม)
- **ไม่ใช่**: ERP (ไม่มีคลังสินค้า/สต็อกเชิงลึก, MRP/การผลิตเต็มรูป, จัดซื้อ, HR/Payroll, BI ข้ามโมดูล)
- **ไม่ใช่**: เครื่องมือ marketing automation/อีเมลแคมเปญขนาดใหญ่ (โฟกัส 1-to-1 sales)
- จุดต่างจาก CRM ต่างประเทศ (HubSpot/Pipedrive/Zoho): **ไทยแท้** — LINE OA, ใบเสนอราคา/ใบกำกับภาษีตามสรรพากร, BahtText, พ.ศ., หัก ณ ที่จ่าย, ราคาแบบ SME ไทย; และ **เข้าใจงานเฟอร์นิเจอร์/งานโครงการ** (จัดกลุ่มห้อง, รูปต่อรายการ, เก็บเงินเป็นงวด, รับประกัน-ติดตั้ง)

### 1.4 เป้าหมายความสำเร็จ (ตัวอย่าง — ปรับได้)
- เซลส์ตอบ lead เร็วขึ้น, อัตราปิดการขายเพิ่มขึ้นวัดได้
- ออกใบเสนอราคา < 5 นาที จากเทมเพลต/แคตตาล็อก
- มีร้านเฟอร์นิเจอร์ SME ใช้งานจริง N ราย ภายในไตรมาส X (กำหนดภายหลัง)

## 2. กลุ่มเป้าหมาย & ผู้ใช้ (Personas)

### 2.1 ลูกค้าของแพลตฟอร์ม (Tenant = ธุรกิจ SME)
- ดีลเลอร์/ตัวแทนจำหน่ายเฟอร์นิเจอร์สำนักงาน (เหมือน anchor)
- โรงงาน/ช่างทำเฟอร์นิเจอร์สั่งทำ-บิวท์อิน
- ร้านเฟอร์นิเจอร์หน้าร้าน/โชว์รูมที่มีงานส่ง-ติดตั้ง
- (ขยายภายหลัง) SME ขายโครงการ/ขายโซลูชันที่ใบเสนอราคาซับซ้อน — ตกแต่งภายใน, ระบบไฟฟ้า/แอร์, ป้าย, ครัวสแตนเลส ฯลฯ
- ขนาด: ~1–30 คน, ส่วนใหญ่จด VAT, อาจใช้สำนักงานบัญชีภายนอก

### 2.2 ผู้ใช้ภายในแต่ละ Tenant
| Persona | ใช้ทำอะไร |
|---|---|
| เจ้าของ/ผู้จัดการ | ดูแดชบอร์ด pipeline/ยอดขาย/ลูกหนี้, ตั้งเป้า, อนุมัติส่วนลด, ตั้งค่าระบบ |
| เซลส์/ที่ปรึกษาการขาย | รับ lead, คุยลูกค้า (LINE/โทร/นัดดูหน้างาน), ออกใบเสนอราคา, ตามต่อ, ปิดดีล |
| แอดมิน/ประสานงาน | บันทึก lead เข้ามาทุกช่องทาง, จัดตารางส่ง-ติดตั้ง, ตามเอกสาร |
| บัญชี/การเงิน (เฟส 2-3) | ออกใบกำกับภาษี/ใบเสร็จ, รับชำระ, ใบลดหนี้/เพิ่มหนี้, รายงานภาษีขาย |
| ช่าง/ทีมติดตั้ง (เฟส 2) | ดูคิวงานติดตั้ง, อัปเดตสถานะ, ให้ลูกค้าเซ็นรับงานในมือถือ |

### 2.3 ผู้เกี่ยวข้องภายนอก
- **ลูกค้าปลายทางของ Tenant** (ฝ่ายจัดซื้อองค์กร / เจ้าของบ้าน): รับใบเสนอราคาทาง LINE/อีเมล, เปิดดู/กดยอมรับออนไลน์, ดูสถานะงาน
- ผู้รับเหมาหลัก (กรณีงานโครงการ): รับ-ส่งเอกสาร, วางบิลเป็นงวด
- สำนักงานบัญชี: รับไฟล์/รายงานภาษีขาย (เฟส 2-3)
- แพลตฟอร์มภายนอก: LINE OA, ผู้ให้บริการอีเมล, โปรแกรมบัญชี (FlowAccount/PEAK/Express) — ผ่าน export/API (เฟสหลัง)

## 3. ขอบเขตและแผนเฟส (Roadmap)

### เฟส 1 — Presale + CRM + ใบเสนอราคา (MVP ที่ขายได้จริง)
- Workspace หลายผู้เช่า (multi-tenant) + สมัครใช้ + ตั้งค่าบริษัท + ผู้ใช้/สิทธิ์
- รับ lead: ฟอร์มเว็บฝังได้ + **LINE OA** + อีเมล (forward-in) + บันทึกโทรเข้า/walk-in ด้วยมือ
- CRM pipeline แบบ Kanban + ดีล/โอกาสการขาย + บันทึกการติดต่อ/งาน/เตือน + ปฏิทิน
- ลูกค้า/ผู้ติดต่อ + Customer 360
- แคตตาล็อกสินค้าแบบเฟอร์นิเจอร์ (หมวด, variant, รูป, ขนาด, วัสดุ, ราคา) + import จาก Excel
- ใบเสนอราคาแบบเฟอร์นิเจอร์: จัดกลุ่มห้อง/โซน + รูปต่อรายการ + ออปชัน + ส่วนลด + VAT 7% + หัก ณ ที่จ่าย (ข้อมูล) + BahtText + เลขที่อัตโนมัติ + revision + workflow อนุมัติส่วนลด
- ส่งใบเสนอราคา: PDF ฟอนต์ไทย + ลิงก์เปิดดูออนไลน์ + ส่งผ่าน LINE/อีเมล + ลูกค้ากดยอมรับ/ปฏิเสธออนไลน์
- Follow-up อัตโนมัติ (ตามหลังส่ง X วัน, เตือนก่อนหมดอายุ)
- Win/Lost + เหตุผลที่แพ้ + เป้าขาย + แดชบอร์ด/รายงาน pipeline
- Responsive/มือถือ, audit log

### เฟส 2 — หลังการขาย: ออเดอร์ → ผลิต → ส่ง → ติดตั้ง → รับประกัน + เอกสาร/ลูกหนี้
- แปลงดีลที่ปิดได้ → ใบสั่งขาย (Sales Order) + บันทึก PO จากลูกค้า
- ติดตามสถานะงาน: สั่งของ/รอวัสดุ → กำลังผลิต → พร้อม → จัดส่ง → ติดตั้งแล้ว (+ กำหนด lead time)
- จัดตารางส่ง/ติดตั้ง + ใบส่งของ + ใบตรวจรับงาน (ลูกค้าเซ็นในมือถือ)
- **ทะเบียนการรับประกัน** (เช่น 1 ปี/รายการ) + เตือนใกล้หมด + บันทึกเคลม/งานแก้
- เก็บเงินเป็นงวด (payment schedule: มัดจำ/ก่อนติดตั้ง/งานเสร็จ) + เงินประกันผลงาน (retention) สำหรับงานโครงการ
- ใบแจ้งหนี้/ใบวางบิล/ใบกำกับภาษี/ใบเสร็จ/ใบลดหนี้/ใบเพิ่มหนี้ + ลูกหนี้การค้า (AR) + AR Aging + ใบแจ้งยอด
- รายงานภาษีขาย + สรุปยอดเพื่อ ภ.พ.30
- เทมเพลตเอกสารหลายแบบ (สองภาษา/อังกฤษ = เฟสหลัง)

### เฟส 3 — บัญชีพื้นฐาน + เชื่อมระบบภายนอก
- ผังบัญชีมาตรฐานไทย + ลงบัญชีอัตโนมัติจากเอกสารขาย/รับเงิน + สมุดรายวัน + บัญชีแยกประเภท + งบทดลอง
- งวดบัญชี + ปิด/ล็อกงวด + งบ P&L / Balance Sheet เบื้องต้น + มิติบัญชี (แผนก/โครงการ)
- API/Webhook + เชื่อม/ส่งออกไปโปรแกรมบัญชีภายนอก (FlowAccount/PEAK/Express/Xero)
- e-Tax Invoice & e-Receipt (กรมสรรพากร)

### นอกขอบเขต (อาจพิจารณาภายหลัง)
วงจรซื้อ/AP เต็มรูป · สต็อก/ต้นทุนสินค้าเชิงลึก · สินทรัพย์ถาวร/ค่าเสื่อม · เงินเดือน/HR · งบกระแสเงินสด/งบรวม · marketing automation/แคมเปญอีเมลขนาดใหญ่ · แอป native (ใช้ responsive web ก่อน) · MRP/วางแผนการผลิต

## 4. ข้อกำหนดเชิงหน้าที่ (Functional Requirements)

### 4.1 Workspace, การสมัคร & การตั้งค่า (Multi-tenant) — เฟส 1
- FR-1.1 สมัครสร้าง workspace ใหม่ได้เอง (ชื่อธุรกิจ, อีเมล, ตั้งรหัสผ่าน) + ทดลองใช้ฟรีช่วงหนึ่ง
- FR-1.2 **แยกข้อมูลแต่ละ tenant เด็ดขาด** (ทุก query ผูก tenant id; ผู้ใช้เห็นเฉพาะ workspace ที่ตนสังกัด; รองรับผู้ใช้อยู่หลาย workspace)
- FR-1.3 ตั้งค่าบริษัท: ชื่อ (ไทย/อังกฤษ), โลโก้, ที่อยู่, **เลขประจำตัวผู้เสียภาษี 13 หลัก**, สำนักงานใหญ่/สาขา, ติดต่อ, บัญชีธนาคาร (หลายบัญชี)
- FR-1.4 ตั้งค่าเริ่มต้น: สกุลเงิน, อัตรา VAT, ภาษาเอกสาร, รูปแบบวันที่ (พ.ศ./ค.ศ.), รูปแบบเลขที่เอกสารต่อชนิด
- FR-1.5 ปรับ pipeline stage, เหตุผล lost, custom field ของ lead/ดีล/ลูกค้า ได้เอง
- FR-1.6 จัดการแพ็ก/บิลลิ่ง: เลือกแพ็ก, จำนวนผู้ใช้, รอบบิล, ใบเสร็จค่าบริการ (ออกให้ผู้ใช้แพลตฟอร์ม) — *(ระดับพื้นฐานในเฟส 1; ปรับละเอียดภายหลัง)*
- FR-1.7 import ข้อมูลตั้งต้น: ลูกค้า, ผู้ติดต่อ, แคตตาล็อกสินค้า จาก Excel/CSV

### 4.2 รับ Lead จากหลายช่องทาง (Lead Capture) — เฟส 1
- FR-2.1 **ฟอร์มขอใบเสนอราคา/ติดต่อ** ที่ฝังในเว็บไซต์ tenant ได้ (embed/ลิงก์) → สร้าง lead อัตโนมัติพร้อมแหล่งที่มา
- FR-2.2 **เชื่อม LINE OA**: ข้อความเข้ามาสร้าง/ผูกกับ lead, แสดงประวัติแชตในโปรไฟล์ lead, ตอบกลับจากในระบบได้
- FR-2.3 **อีเมลขาเข้า**: forward อีเมลสอบถามเข้า address ของ workspace → สร้าง lead + แนบเนื้อหา
- FR-2.4 บันทึก lead ด้วยมือ: โทรเข้า, walk-in โชว์รูม, งานออกบูธ, แนะนำต่อ (referral)
- FR-2.5 (ออปชันเฟสหลัง) Facebook/Messenger lead form, เว็บแชต
- FR-2.6 กันซ้ำ (dedupe) ตามเบอร์/อีเมล/ชื่อ + รวม lead ซ้ำได้
- FR-2.7 มอบหมาย lead ให้เซลส์ (กติกาอัตโนมัติ เช่น round-robin/ตามพื้นที่ หรือมือ) + แจ้งเตือน

### 4.3 CRM Pipeline & ดีล/โอกาสการขาย — เฟส 1
- FR-3.1 มุมมอง **Kanban** ตาม stage (ปรับ stage ได้): ลูกค้าใหม่ → คัดกรอง/Qualify → ส่งใบเสนอราคา → ต่อรอง → ปิดได้ / ปิดไม่ได้ — ลากการ์ดเปลี่ยน stage
- FR-3.2 ข้อมูลดีล: ชื่อดีล/โครงการ, ลูกค้า/ผู้ติดต่อ, เซลส์เจ้าของ, มูลค่าโดยประมาณ, % โอกาสปิด, วันคาดปิด, แหล่งที่มา/ช่องทาง, สินค้าที่สนใจ (หมวด/รุ่น), งบลูกค้า, แท็ก
- FR-3.3 **Timeline กิจกรรมในดีล**: บันทึกการโทร/แชต LINE/อีเมล/นัดดูหน้างาน/หมายเหตุ ทั้งหมดเรียงตามเวลา + แนบไฟล์/รูป
- FR-3.4 เปลี่ยนเป็นมุมมองตาราง/ลิสต์ + กรอง/ค้นหา/เรียง
- FR-3.5 ปิดดีล: Won (ผูกใบเสนอราคาที่ลูกค้าตอบรับ → ไปสร้างใบสั่งขายเฟส 2) / Lost (เลือกเหตุผล: ราคาสูง/lead time นาน/คู่แข่ง/งบไม่พอ/เลื่อนโครงการ/อื่น ๆ + หมายเหตุ)
- FR-3.6 reopen ดีลที่ปิดไปแล้วได้ (พร้อมเหตุผล)

### 4.4 ลูกค้า / ผู้ติดต่อ / Customer 360 — เฟส 1
- FR-4.1 ลูกค้า (บริษัท/บุคคล): ชื่อ (ไทย/อังกฤษ), ประเภท, ที่อยู่ออกใบกำกับภาษี + ที่อยู่จัดส่ง/หน้างาน, **เลขผู้เสียภาษี**, สาขา, เงื่อนไขเครดิตเริ่มต้น, แท็ก
- FR-4.2 ผู้ติดต่อหลายคนต่อลูกค้า (ชื่อ, ตำแหน่ง, แผนก, โทร, อีเมล, LINE) — รองรับเคส "ฝ่ายจัดซื้อ" แยกจาก "ผู้ใช้งานจริง/หน้างาน"
- FR-4.3 **Customer 360**: หน้ารวม lead/ดีล/ใบเสนอราคา/ออเดอร์/การส่ง-ติดตั้ง/การรับประกัน/ใบแจ้งหนี้-ลูกหนี้/ประวัติการติดต่อ ของลูกค้ารายนั้น
- FR-4.4 ค้นหา/กรอง, soft delete/archive (ห้ามลบถ้ามีเอกสารผูก), import/export

### 4.5 กิจกรรม งาน เตือนความจำ & ปฏิทิน — เฟส 1
- FR-5.1 สร้าง task ผูกกับ lead/ดีล/ลูกค้า: ประเภท (โทรกลับ/ส่งตัวอย่าง/นัดสำรวจหน้างาน/ส่งใบเสนอราคา/ติดตาม), กำหนดวัน-เวลา, มอบหมายผู้รับผิดชอบ
- FR-5.2 รายการงานของฉัน/ของทีม + งานเกินกำหนด (overdue) + แจ้งเตือน (ในระบบ/อีเมล/—ออปชัน push)
- FR-5.3 ปฏิทินรวม (นัดหมาย, งานติดตั้ง, ส่งของ) — มุมมองวัน/สัปดาห์/เดือน; (ออปชันเฟสหลัง) sync Google Calendar
- FR-5.4 บันทึก log การติดต่อด่วน (quick log: โทรแล้ว/ส่ง LINE แล้ว + โน้ตสั้น) จากการ์ดดีลโดยตรง

### 4.6 แคตตาล็อกสินค้า (เน้นเฟอร์นิเจอร์) — เฟส 1
- FR-6.1 หมวดสินค้าหลายระดับ (เช่น โต๊ะ > โต๊ะผู้บริหาร / โต๊ะทำงาน; เก้าอี้; ตู้เซฟ; ตู้รางเลื่อน; ล็อกเกอร์; ชุดประชุม; โซฟา)
- FR-6.2 รายการสินค้า: รหัส, ชื่อ (ไทย/อังกฤษ), รายละเอียด, **รูปหลายรูป/3D**, หน่วยนับ, ราคาขายเริ่มต้น, ต้นทุน (ออปชัน — ดู §4.10), แท็ก
- FR-6.3 **คุณสมบัติเฉพาะเฟอร์นิเจอร์**: ขนาด W×D×H, วัสดุ/ผิว (ไม้/ลามิเนต/เมลามีน/วีเนียร์/เหล็ก), สี/รหัสสี, ยี่ห้ออุปกรณ์ (บานพับ/รางลิ้นชัก), น้ำหนัก, มาตรฐาน (เช่น มอก. สำหรับตู้เซฟ)
- FR-6.4 **Variant**: รุ่นเดียวกันหลายขนาด/สี/วัสดุ → ราคาต่างกัน
- FR-6.5 **รายการแบบชุด/บันเดิล** (เช่น "ชุดโต๊ะประชุม 12 ที่นั่ง" = โต๊ะ + เก้าอี้ × 12) — กาง/ยุบในใบเสนอราคาได้
- FR-6.6 **ออปชัน/add-on ต่อรายการ** (เช่น เพิ่มไฟ LED ใต้ตู้, เพิ่มลิ้นชักเก็บของ, อัปเกรดเบาะหนังแท้ +ราคา)
- FR-6.7 ราคาตามลูกค้า/ระดับลูกค้า (ออปชัน), ราคาขั้นบันได
- FR-6.8 ค้นหา/กรองตามหมวด/วัสดุ/ช่วงราคา, import/export, เปิด/ปิดการขายรายการ

### 4.7 ใบเสนอราคา (Furniture-aware Quotation) — เฟส 1
**หัวเอกสาร**
- FR-7.1 **เลขที่เอกสารอัตโนมัติ** — รูปแบบกำหนดเองได้ (`QT-{YYYY}-{####}` หรือ พ.ศ. `QT-{BBBB}-{####}`), รันต่อเนื่อง ไม่ซ้ำ, รีเซ็ตตามปี/สาขาได้
- FR-7.2 วันที่ออก (default วันนี้), **ยืนราคาถึง / Valid until** (default +30 วัน, ตั้งค่าได้)
- FR-7.3 เลือกลูกค้า + ผู้ติดต่อ (auto-fill ที่อยู่/เลขผู้เสียภาษี/เงื่อนไข); ผูกกับดีลใน pipeline
- FR-7.4 อ้างอิง (ชื่อโครงการ/หน้างาน, เลข RFQ/PR ของลูกค้า), เซลส์ผู้รับผิดชอบ, สกุลเงิน, เทมเพลต — *(ภาษาเอกสาร: ไทยอย่างเดียวในเฟส 1; ตัวเลือกภาษา/สองภาษาเป็นเฟสหลัง)*

**รายการในเอกสาร — แบบเฟอร์นิเจอร์**
- FR-7.5 **จัดกลุ่มตามห้อง/โซน/ชั้น** (เช่น "ชั้น 3 - ห้องผู้บริหาร", "Lobby", "ห้องประชุมใหญ่") — มี subtotal ต่อกลุ่ม, ย้าย/จัดเรียงกลุ่มและรายการได้ (drag)
- FR-7.6 บรรทัดรายการ: ลำดับ, รหัส/รุ่น, ชื่อ+รายละเอียด (แก้ไขได้), **ขนาด W×D×H**, วัสดุ/สี/ผิว, **รูปประกอบ 1+ รูป**, จำนวน, หน่วย, ราคา/หน่วย, ส่วนลดต่อบรรทัด (บาท/%), จำนวนเงิน
- FR-7.7 ดึงจากแคตตาล็อก (พร้อมรูป/คุณสมบัติ/ออปชัน) หรือพิมพ์รายการอิสระ (free text + แนบรูปเองได้)
- FR-7.8 รายการแบบชุด (กาง/ยุบ), ออปชัน/add-on ต่อรายการ (เลือกแล้วคิดเงินเพิ่ม)
- FR-7.9 บรรทัดพิเศษ: หัวข้อ/กลุ่ม (heading), ข้อความ (note), ส่วนลดท้ายบิล, **ค่าจัดส่ง**, **ค่าติดตั้ง**
- FR-7.10 รองรับหลายประเภทภาษีในใบเดียว (VAT 7% / 0% / ยกเว้น), ราคาแบบ exclusive/inclusive VAT

**การคำนวณ**
- FR-7.11 รวมเงิน → ส่วนลดท้ายบิล → มูลค่าหลังหักส่วนลด → แยกฐาน VAT แต่ละอัตรา → VAT → ยอดรวมทั้งสิ้น
- FR-7.12 แสดงประมาณการ **ภาษีหัก ณ ที่จ่าย** ที่ลูกค้าจะหัก (ตามอัตราต่อบรรทัด/ต่อใบ) เป็นข้อมูล + ยอดคาดรับสุทธิ
- FR-7.13 **จำนวนเงินรวมเป็นตัวอักษรภาษาไทย (BahtText)** จัดการ "บาทถ้วน"/"สตางค์" ถูกต้อง (เวอร์ชันอังกฤษ = เฟสหลัง)
- FR-7.14 การปัดเศษกำหนดได้ (2 ตำแหน่ง, half-up), แสดงบรรทัด "ปัดเศษ" ถ้ามี
- FR-7.15 (ออปชัน) แสดง/ซ่อน **margin ต่อรายการ/ต่อใบ** สำหรับผู้มีสิทธิ์ (ดู §4.10)

**ท้ายเอกสาร**
- FR-7.16 เงื่อนไขการชำระเงิน — เลือก preset หรือพิมพ์เอง; รองรับ **payment schedule หลายงวด** (เช่น มัดจำ 50% / ก่อนติดตั้ง 30% / งานเสร็จ 20%) แสดงบนเอกสาร
- FR-7.17 กำหนดส่งมอบ/ติดตั้ง (lead time เช่น 30-45 วันหลังยืนยันและรับมัดจำ), การรับประกัน (เช่น 1 ปี), เงื่อนไขอื่น/หมายเหตุ, เลขบัญชีธนาคาร
- FR-7.18 ช่องลายเซ็น 3 ฝ่าย: ผู้เสนอราคา / ผู้มีอำนาจอนุมัติ (ฝั่ง tenant) / ผู้อนุมัติสั่งซื้อ (ฝั่งลูกค้า) + วันที่; รองรับรูปลายเซ็น/ตราประทับ

**การจัดการเอกสาร**
- FR-7.19 บันทึกร่าง (auto-save), ยืนยัน, คัดลอกเป็นใบใหม่, สร้างจากเทมเพลต/จากใบเดิม
- FR-7.20 **Revision**: แก้ใบที่ส่งแล้ว → สร้างเวอร์ชันใหม่ (Rev.1, 2, ...), เก็บทุกเวอร์ชัน + เหตุผล + เปรียบเทียบความต่าง; ระบุชัดบนเอกสารและในไทม์ไลน์ดีลว่าส่ง Rev. ไหน
- FR-7.21 workflow อนุมัติภายใน: ส่วนลด/มูลค่าเกิน threshold ต้องให้ผู้มีสิทธิ์อนุมัติก่อนส่ง
- FR-7.22 ยกเลิกใบ (ระบุเหตุผล) — ไม่ลบทิ้ง; แนบไฟล์เพิ่ม (สเปก/แบบ/รูปหน้างาน)
- FR-7.23 สถานะใบเสนอราคา: ร่าง → รออนุมัติ → พร้อมส่ง → ส่งแล้ว → ลูกค้าตอบรับ / ปฏิเสธ / หมดอายุ / ยกเลิก; อัปเดต "หมดอายุ" อัตโนมัติเมื่อเลยวันยืนราคา

### 4.8 ส่งเอกสาร & Follow-up — เฟส 1
- FR-8.1 สร้าง **PDF ฟอนต์ไทยฝังในไฟล์** (เช่น Sarabun/TH Sarabun New); A4; หัว/ท้ายกระดาษ; เลขหน้า x/y; โลโก้; ลายน้ำ ("ตัวอย่าง"/"ยกเลิก"); แสดงรูปสินค้าในตารางรายการ
- FR-8.2 หลายเทมเพลต + ปรับสี/โลโก้/ฟิลด์ที่แสดง (เอกสารสองภาษาในไฟล์เดียว = เฟสหลัง)
- FR-8.3 ส่งให้ลูกค้า: **แชร์ลิงก์เปิดดูออนไลน์** (มีวันหมดอายุ) / ส่งทาง **LINE** / อีเมล (แนบ PDF + เทมเพลตข้อความไทย)
- FR-8.4 ลูกค้าหน้าเว็บ (ไม่ต้องล็อกอิน): ดูใบเสนอราคา, ดาวน์โหลด PDF, **กด "ยอมรับ"/"ขอแก้ไข"/"ปฏิเสธ"** พร้อมใส่ชื่อ-ลายเซ็นอิเล็กทรอนิกส์ + บันทึก IP/เวลา → อัปเดตสถานะดีลอัตโนมัติ
- FR-8.5 บันทึกประวัติส่ง (เมื่อไหร่ ถึงใคร ช่องทางไหน Rev. ไหน) + แจ้งเตือนเมื่อลูกค้าเปิดดู
- FR-8.6 **Follow-up อัตโนมัติ**: สร้าง task/แจ้งเซลส์ตามหลังส่ง X วันถ้าลูกค้ายังไม่ตอบ, เตือนก่อนใบหมดอายุ Y วัน — ตั้งค่า cadence ได้

### 4.9 Win/Loss, เป้าขาย, แดชบอร์ด & รายงาน — เฟส 1
- FR-9.1 แดชบอร์ด pipeline: มูลค่าตาม stage, ดีลที่ค้างนานผิดปกติ, งานที่ต้องตามวันนี้, ใบเสนอราคาใกล้หมดอายุ
- FR-9.2 รายงานยอดขาย/ดีล: ตามเซลส์ / เดือน / สินค้า-หมวด / ลูกค้า / แหล่งที่มา-ช่องทาง
- FR-9.3 อัตราการปิดการขาย (Won / รวม), เวลาเฉลี่ยจนปิด, มูลค่า forecast ถ่วงน้ำหนัก %
- FR-9.4 วิเคราะห์ Lost: สัดส่วนเหตุผลที่แพ้, แพ้ให้คู่แข่งรายไหนบ่อย
- FR-9.5 **เป้าขายรายเซลส์/รายทีม/รายเดือน** + ทำได้เทียบเป้า
- FR-9.6 ส่งออกรายงานเป็น Excel/CSV/PDF

### 4.10 ต้นทุน/กำไร & ราคา supplier (ออปชัน — เฟส 1/2)
- FR-10.1 บันทึกต้นทุนต่อรายการในแคตตาล็อก (และต้นทุนเฉพาะใบเสนอราคาได้)
- FR-10.2 แสดง **margin / กำไรขั้นต้น** ต่อบรรทัด/ต่อกลุ่มห้อง/ต่อใบ — เฉพาะผู้มีสิทธิ์ (เจ้าของ/ผู้จัดการ); ซ่อนจากเอกสารที่ส่งลูกค้าเสมอ
- FR-10.3 (งานโครงการ) บันทึก **ใบขอราคาจาก supplier วัสดุ** แล้วดึงต้นทุนมาคำนวณราคางาน

### 4.11 หลังการขาย: ออเดอร์ → ผลิต → ส่ง → ติดตั้ง — เฟส 2
- FR-11.1 แปลงดีล Won/ใบเสนอราคาที่ตอบรับ → **ใบสั่งขาย (Sales Order)** + บันทึก **PO ของลูกค้า** (เลข PO, วันที่, ไฟล์แนบ)
- FR-11.2 รองรับแปลงบางส่วน (partial), เชื่อมโยงอ้างอิงทั้งสายโซ่ (ดีล ↔ ใบเสนอราคา ↔ SO ↔ ใบส่งของ ↔ ใบแจ้งหนี้)
- FR-11.3 สถานะงาน (กำหนดเองได้): รับ PO → สั่งวัสดุ/รอของ → กำลังผลิต → ผลิตเสร็จ → จัดส่ง → ติดตั้งแล้ว → ปิดงาน; ระบุ lead time/วันคาดเสร็จ; แจ้งเตือนเมื่อเลยกำหนด
- FR-11.4 **จัดตารางส่ง/ติดตั้ง**: นัดวัน-เวลา, มอบหมายทีมช่าง, ที่อยู่หน้างาน, รายการที่จะส่ง/ติดตั้ง
- FR-11.5 **ใบส่งของ** + **ใบตรวจรับงาน/ส่งมอบ** ให้ลูกค้าเซ็นรับ (รองรับเซ็นบนมือถือ + ถ่ายรูปงานหน้างาน)
- FR-11.6 แจ้งเตือน/อัปเดตสถานะให้ลูกค้าผ่าน LINE/อีเมล (ออปชัน)

### 4.12 ทะเบียนการรับประกัน & เคลม — เฟส 2
- FR-12.1 เมื่อส่งมอบงาน → สร้าง **การรับประกัน** อัตโนมัติต่อรายการ/ต่อออเดอร์ (ระยะเช่น 1 ปี, วันเริ่ม = วันส่งมอบ, วันหมด)
- FR-12.2 ค้นหาการรับประกันตามลูกค้า/สินค้า/serial; แจ้งเตือนใกล้หมดประกัน
- FR-12.3 บันทึก **เคลม/งานแก้**: อ้างการรับประกัน/ออเดอร์เดิม, อาการ, นัดช่าง, สถานะ (รับเรื่อง → นัด → แก้ไข → ปิด), ค่าใช้จ่าย (ในประกัน/นอกประกัน)
- FR-12.4 (ออปชัน) ออกใบรับประกัน PDF ให้ลูกค้า

### 4.13 ใบกำกับภาษี / ใบเสร็จ / ใบลดหนี้-เพิ่มหนี้ + ลูกหนี้ (AR) — เฟส 2
- FR-13.1 แปลง SO/ใบเสนอราคา → **ใบแจ้งหนี้/ใบวางบิล** → **ใบกำกับภาษีแบบเต็มรูป** (ครบตาม §5.2, ระบุ "ต้นฉบับ/สำเนา", immutable หลังออก) → **ใบเสร็จรับเงิน**
- FR-13.2 **ใบรับเงินมัดจำ** จาก payment schedule; ตัดมัดจำเมื่อออกใบงวดถัดไป; รองรับ **เงินประกันผลงาน (retention)** หักไว้ % แล้วออกใบเรียกเก็บคืนภายหลัง
- FR-13.3 **ใบลดหนี้/ใบเพิ่มหนี้** อ้างใบกำกับภาษีเดิม + เหตุผลตามกฎหมาย; ปรับยอดลูกหนี้และฐานภาษี
- FR-13.4 **รับชำระเงิน**: วันที่, ช่องทาง (เงินสด/โอน/เช็ค/บัตร), บัญชีที่รับ, จำนวน, ค่าธรรมเนียม, **ภาษีหัก ณ ที่จ่ายที่ลูกค้าหัก** (อ้างหนังสือรับรองฯ), จับคู่กับใบแจ้งหนี้ (เต็ม/บางส่วน/หลายใบ), เงินรับล่วงหน้า, ตัดหนี้สูญ
- FR-13.5 ลูกหนี้คงค้างต่อใบ/ต่อลูกค้า, วันครบกำหนด, **AR Aging** (ยังไม่ครบ/1-30/31-60/61-90/>90), **ใบแจ้งยอดลูกค้า** + ส่งอีเมล/LINE, แจ้งเตือนหนี้ใกล้/เกินกำหนด, จัดการเช็ครับ (รอเก็บ/ผ่าน/คืน)

### 4.14 ภาษีขาย & บัญชีพื้นฐาน — เฟส 2-3
- FR-14.1 (เฟส 2) **รายงานภาษีขาย** ตามรูปแบบสรรพากร (เลขที่ใบกำกับภาษี, ผู้ซื้อ-เลขผู้เสียภาษี, มูลค่า, VAT) ต่อเดือนภาษี + สรุปยอดเพื่อ ภ.พ.30; รายงานสรุปภาษีหัก ณ ที่จ่ายที่ถูกลูกค้าหัก; จัดการเลขที่ที่ยกเลิก (คงเลข + ระบุ "ยกเลิก"); ส่งออก Excel/PDF ให้สำนักงานบัญชี
- FR-14.2 (เฟส 3) ผังบัญชีมาตรฐานไทย + ปรับแต่งได้
- FR-14.3 (เฟส 3) **ลงบัญชีอัตโนมัติ** จากเอกสาร: ออกใบกำกับภาษี → Dr ลูกหนี้ / Cr รายได้, ภาษีขาย; รับชำระ → Dr เงินสด-ธนาคาร, ภาษีหัก ณ ที่จ่าย / Cr ลูกหนี้; ใบลดหนี้/เพิ่มหนี้ → กลับรายการตามสัดส่วน — mapping บัญชีปรับได้
- FR-14.4 (เฟส 3) สมุดรายวัน (รวม manual JV), บัญชีแยกประเภท, งบทดลอง
- FR-14.5 (เฟส 3) งวดบัญชี + ปิด/ล็อกงวด + ยอดยกมา; งบ P&L / Balance Sheet เบื้องต้น + เปรียบเทียบงวด; มิติบัญชี (แผนก/โครงการ/ศูนย์ต้นทุน — ติด tag บนบรรทัดเอกสารตั้งแต่เฟส 1-2)
- FR-14.6 *(ยังไม่รวม)* วงจรซื้อ/AP, สต็อก/ต้นทุนสินค้า, สินทรัพย์ถาวร/ค่าเสื่อม, งบกระแสเงินสด, ภ.พ.30/ภ.ง.ด. ยื่นออนไลน์, งบรวม

### 4.15 ผู้ใช้ สิทธิ์ & ความปลอดภัย — เฟส 1
- FR-15.1 บัญชีผู้ใช้ + ล็อกอิน (รหัสผ่าน + ตัวเลือก 2FA); เชิญผู้ใช้เข้า workspace ทางอีเมล
- FR-15.2 บทบาทต่อ workspace: เจ้าของ/แอดมิน / ผู้จัดการ-ผู้อนุมัติ / เซลส์ / บัญชี / ช่าง / ผู้ดูอย่างเดียว — กำหนดสิทธิ์ระดับฟังก์ชันและชนิดเอกสาร
- FR-15.3 จำกัดข้อมูลที่เห็น (เช่น เซลส์เห็นเฉพาะ lead/ดีลของตน — ตั้งค่าได้); จำกัดการเห็น margin/ต้นทุน
- FR-15.4 วงเงิน/อัตราส่วนลดที่อนุมัติได้ ต่อบทบาท/ผู้ใช้
- FR-15.5 **Audit log** ทุกการสร้าง/แก้ไข/ลบ/เปลี่ยนสถานะ/ส่งเอกสาร/รับเงิน/ออกใบกำกับภาษี/ส่งออก (ใคร เมื่อไหร่ ค่าก่อน-หลัง)

### 4.16 การเชื่อมต่อ (Integrations) — เฟส 1+ (ทยอย)
- FR-16.1 **LINE Official Account** — รับ-ส่งข้อความ, ส่งเอกสาร, แจ้งเตือนลูกค้า (เฟส 1 ระดับพื้นฐาน)

## 5. ข้อกำหนดด้านกฎหมาย/ภาษีของไทย

### 5.1 สถานะทางกฎหมายของใบเสนอราคา
- ใบเสนอราคา **ไม่ใช่เอกสารทางภาษีอากร** จึงไม่มีรูปแบบบังคับ — แต่ต้องออกแบบให้แปลงไปเป็นใบกำกับภาษีแบบเต็มรูปได้โดยไม่กรอกข้อมูลซ้ำ

### 5.2 ใบกำกับภาษีแบบเต็มรูป (มาตรา 86/4) — เมื่อแปลงเอกสาร (เฟส 2) ต้องมีครบ
1. คำว่า **"ใบกำกับภาษี"** เห็นเด่นชัด
2. ชื่อ ที่อยู่ และ **เลขประจำตัวผู้เสียภาษีอากร** ของผู้ออก
3. ชื่อ ที่อยู่ ของผู้ซื้อ (และเลขผู้เสียภาษีหากเป็นนิติบุคคล/ผู้ประกอบการ)
4. หมายเลขลำดับของใบกำกับภาษี (และเล่มที่ ถ้ามี)
5. ชื่อ ชนิด ประเภท ปริมาณ และมูลค่าของสินค้า/บริการ
6. จำนวน **ภาษีมูลค่าเพิ่มแยกออกจากมูลค่าสินค้า/บริการอย่างชัดแจ้ง**
7. วัน เดือน ปี ที่ออกใบกำกับภาษี
8. ระบุ **"สำนักงานใหญ่"** หรือ **"สาขาที่ ..."** ของทั้งผู้ขายและผู้ซื้อ (ถ้าจดทะเบียนแบบมีสาขา)

### 5.3 ภาษีมูลค่าเพิ่ม (VAT)
- อัตรามาตรฐาน **7%** (ตั้งค่าได้); รองรับแยกฐานในใบเดียว: VAT 7% / VAT 0% (เช่นส่งออก) / ยกเว้น; รองรับราคา exclusive และ inclusive VAT

### 5.4 ภาษีหัก ณ ที่จ่าย (Withholding Tax)
- ใบเสนอราคา: แสดงเป็น **ข้อมูลประกอบ** (ไม่หักออกจากยอด); ตอนรับชำระเงิน (เฟส 2): บันทึกยอดที่ลูกค้าหักจริง อ้างหนังสือรับรองฯ
- รองรับอัตราที่พบบ่อย: 1% (ค่าขนส่ง), 2% (โฆษณา), 3% (บริการ/รับจ้างทำของ — ครอบคลุมงานติดตั้ง/งานทำเฟอร์นิเจอร์สั่งทำ), 5% (ค่าเช่า) — ตั้งค่าได้

### 5.5 อื่น ๆ
- จัดเก็บเอกสาร/หลักฐาน โดยทั่วไป ≥ 5 ปี
- แสดงปี **พ.ศ. และ ค.ศ.** (ค่าเริ่มต้นบนเอกสารไทย = พ.ศ.)
- **PDPA**: เก็บข้อมูลส่วนบุคคลของผู้ติดต่อ/ลูกค้า ต้องมีฐานการประมวลผลที่ชอบด้วยกฎหมาย, จำกัดสิทธิ์การเข้าถึง, รองรับคำขอใช้สิทธิ์ของเจ้าของข้อมูล (เข้าถึง/แก้ไข/ลบ/ส่งออก); ในฐานะแพลตฟอร์ม SaaS = ผู้ประมวลผลแทน tenant → ต้องมีสัญญา/ข้อตกลงประมวลผลข้อมูล (DPA)
- เลขที่เอกสารภาษี (ใบกำกับภาษี/ใบลดหนี้/เพิ่มหนี้) ต้องเรียงต่อเนื่อง ไม่มีช่องว่าง; ที่ยกเลิกต้องคงเลขไว้ในรายงานพร้อมระบุ "ยกเลิก"

## 7. ข้อกำหนดที่ไม่ใช่เชิงหน้าที่ (Non-Functional Requirements)

| ด้าน | ข้อกำหนด |
|---|---|
| Multi-tenancy | แยกข้อมูลแต่ละ tenant เด็ดขาด; ทดสอบ data leakage; จำกัดทรัพยากรต่อ tenant; ลบข้อมูลเมื่อ tenant ปิดบัญชี (ตามนโยบาย) |
| Custom domains | แต่ละ tenant ใช้ได้ทั้ง subdomain ในตัว (`<slug>.<app-domain>`) และ **โดเมนของตัวเอง** (เช่น crm.example.com) — ต้องมีขั้นตอนยืนยันความเป็นเจ้าของโดเมน + ออก TLS อัตโนมัติให้โดเมนนั้น (on-demand TLS); resolve tenant จาก host ของ request |
| ภาษา/Localization | **MVP: ไทยอย่างเดียว** (UI + เอกสาร); วันที่ พ.ศ./ค.ศ.; ตัวเลขคั่นหลักพัน; สกุลเงิน — สองภาษา (ไทย/อังกฤษ) = เฟสหลัง |
| Timezone | Asia/Bangkok (UTC+7) เป็นค่าเริ่มต้น (ตั้งต่อ tenant ได้) |
| มือถือ | **Responsive ตั้งแต่เฟส 1** — ใช้บนมือถือ/แท็บเล็ตได้จริง (ดู pipeline, ออก/ส่งใบเสนอราคา, เซ็นรับงานหน้างาน) |

## 10. คำถามที่ต้องเคลียร์ (Open Questions)

4. โฮสต์/ภูมิภาคข้อมูล: ต้องเก็บข้อมูลในไทย/อาเซียนไหม (ผลด้านความน่าเชื่อถือ/PDPA)?
5. Tech stack ที่ต้องการ/มีอยู่ (ภาษา, framework, ฐานข้อมูล, คลาวด์)? มีทีม dev กี่คน?
6. งบ/ไทม์ไลน์คร่าว ๆ ของเฟส 1?
7. ชื่อผลิตภัณฑ์/แบรนด์ — มีในใจหรือยัง?
8. ใช้สำนักงานบัญชีภายนอกอยู่ไหม — รูปแบบรายงาน/ไฟล์ที่เขาต้องการรับเป็นแบบไหน (เผื่อเฟส 2)?
